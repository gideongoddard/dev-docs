Git

We'll cover the basic git commands that you would use for version control using a local repository, then cover the commands that you'd need when working with branches and finally commands relating to working with remote repositories.

Getting started

Configuring git

git config

Git adds an Author attribute to every commit you make, so two aspects of configuration that are essential for it to function are:

  1. git config --global user.name "Your Name"
  2. git config --global user.email "your.email@example.com"

Creating a local repository

git init

This command should be used when the working directory is the project root. No options or arguments are necessary. This is used for starting a new project.

Cloning an existing repository

git clone

This is used when your starting point for a project is a pre-existing repository and you need to create a local copy of it. The command will be used with the location of the remote repository that you're cloning - for example:

git clone https://clientco.co/our-website.git

The basics

Checking git status

git status

Adding files to your staging area

git add

This command can be used with a single file name, multiple file names, single or multiple directory names or with an an asterisk wildcard for adding everything. Here are some examples:

  • git add index.html
  • git add index.html styles.css
  • git add src/
  • git add *

Committing changes

git commit
