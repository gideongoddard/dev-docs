The command line

We'll cover commands that you'll need to know for navigating and manipulating the file system. There will be many other ways that you might want to use the command line for other specific objectives but they'll be covered under more specific subject areas.

Navigating the file system

Print working directory

pwd

This command is used on its own without any options. It will give you the complete path for the working directory.

List contents of working directory

ls

This command can be used on its own or with other options. On its own, it will list the contents of the working directory. Other options are as follows:

  1. ls -a This lists all contents, including hidden files and folders.
  2. ls -l This lists the contents of the working directory in long format.
  3. ls -t This lists the contents of the working directory by the time they were last edited.

Change directory

cd

This command is always used with the relative path that you wish to navigate to. Some examples are:

  • cd child-directory This assumes there is a folder within the working directory called 'child-directory'. The command will change to this directory.
  • cd .. This navigates to the parent directory of the current working directory using '..' as shorthand for going up one level.
  • cd ../feet/right-foot This navigates up one level and then into a 'feet' directory and then into a 'right-foot' directory.

Creating files and folders

Make directory

mkdir

This command is always used with the name of the directory you wish to create. Note that the directory will be created within the working directory and that if there are any spaces in the directory name then the name must be in quotation marks.

Create new file

touch

This command is essentially identical to the 'mkdir' command above, except that it will create a file rather than a folder. This means that an appropriate filetype extension should be applied along with the filename.

Moving and renaming

Copy

cp

The copy command can either copy one or more files. The command is used along with at least two options - the file(s) that are being copied and the destination. Some examples:

  • cp file-one.txt file-two.txt This copies the contents of 'file-one.txt' into 'file-two.txt' where they are both in the working directory.
  • cp animals/shark.txt fish/ This copies the file 'shark.txt' from the 'animals' directory into the 'fish' directory.
  • cp animals/crow.txt animals/eagle.txt birds/ This copies both 'crow.txt' and 'eagle.txt' files into the 'birds' directory. Note that both this example and the one above assumes that you are currently in the parent directory of 'animals', 'fish' and 'birds' given the relative file paths.
  • cp * ../comedy/ This assumes that you are currently in a directory named 'satire' and you are copying all of the contents into another directory called 'comedy' at the same level in the filesystem.
  • cp m*.txt scifi/ This assumes that you are currently in a directory named 'action' and you are copying all files beginning with the letter 'm' into another directory called 'scifi' which is a sub-directory of 'action'.

Move

mv

This command can be used for both moving a file and renaming a file. When moving files, the 'mv' command is similar to the 'cp' command - akin to 'cut' and paste vs. 'copy' and paste.

To move files, the command is used with one or more file names as the first argument and a destination directory as the last argument:

mv matrix.txt matrix-2.txt matrix-3.txt scifi/

To rename a file, the command is used with one existing file name as the first argument and the new file name as the last argument:

mv matrix-2.txt matrix-reloaded.txt
